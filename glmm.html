Extension · Generalised Linear Mixed Models

When Y Isn't
Normal

LMMs assume your outcome is normally distributed. But remission (yes/no) is binary. PHQ-8 is a count. Severity ratings are ordinal. GLMMs extend LMMs to any outcome type — by adding one extra layer: a link function.

1
The Problem with LMM for Non-Normal Outcomes

What happens when you use LMM on binary data

Suppose your outcome is whether a patient is in remission (1 = yes, 0 = no). If you fit a standard LMM, the model will happily predict values like 1.4 or −0.3 — which are nonsensical probabilities. The linear predictor Xβ + Zu lives on (−∞, +∞), but probabilities live on [0, 1]. You need something to map between these two spaces.

// Figure: LMM vs GLMM for binary outcome — why LMM predictions go out of bounds
Left: LMM fitted line passes below 0 and above 1 — impossible probabilities. Right: GLMM logistic curve stays bounded within [0,1] everywhere. The S-shape reflects the logistic link function mapping the linear predictor back to probability space.

Wait — is PHQ-8 a count?

PHQ-8 is technically an ordinal sum score (0–24). In practice, researchers often treat it as approximately continuous and use LMM — especially after confirming residuals look roughly normal. This is a pragmatic choice. If PHQ-8 values are very skewed (many zeros, few highs) or your sample is small, a GLMM with an appropriate family is more rigorous.

3
The Link Function

What a link function actually does

The GLMM equation from Wikipedia is: g(E[y|u]) = Xβ + Zu. This says: apply function g() to the expected value of y (given random effects u), and that equals the familiar linear predictor. The link function g() is a one-way bridge between the natural scale of y and the unconstrained linear predictor.

// Three common link functions — what they do and why
Outcome scale
Probability p ∈ [0, 1]
logit link

g(p) = log(p/1−p)
Linear predictor scale
log-odds ∈ (−∞, +∞)

The logit of a probability is the log of the odds ratio: log(p / (1−p)). A probability of 0.5 maps to logit = 0. A probability of 0.9 maps to logit = 2.2. A probability of 0.01 maps to logit = −4.6. The logit can be any real number — so the linear predictor (Xβ + Zu) can take any value, and you invert g to get back to probability: p = 1/(1+exp(−Xβ−Zu)). This is the sigmoid/logistic function.

Outcome scale
Count λ ∈ [0, ∞)
log link

g(λ) = log(λ)
Linear predictor scale
log-rate ∈ (−∞, +∞)

The log link ensures the model never predicts negative counts. The linear predictor Xβ + Zu lives on (−∞, +∞), but exp(Xβ + Zu) is always positive. Coefficients from a log-link model are interpreted as multiplicative effects on the rate: eᵝ is the rate ratio.

// Figure: The logit link — mapping probability [0,1] ↔ log-odds (−∞,+∞)
Left panel: the sigmoid/logistic function — mapping any linear predictor value to a probability. Right panel: the logit function (its inverse) — mapping any probability to the unconstrained log-odds scale. The S-shape ensures the model stays in [0,1] even when Xβ+Zu is very large or very small.
4
The Hard Part: No Closed Form

Why GLMMs are computationally harder than LMMs

For a standard LMM with normal outcomes, the random effects u can be integrated out analytically — there's a clean closed-form solution. For GLMMs, this is no longer possible.

The Wikipedia equation for the marginal likelihood (the probability of the data, having integrated out the random effects) is:

// The likelihood we want to maximise:
p(y) = ∫ p(y|u) · p(u) du

// What each piece means:
p(y|u) = probability of data given random effects = GLM likelihood
p(u) = probability of random effects = N(0, G) (normal prior on u)
∫ ... du = sum over ALL possible values of u — an infinite-dimensional integral
// Figure: Why the integral is hard — visualising ∫ p(y|u) · p(u) du
What this integral is doing: For every possible value of the random effect u (x-axis), we compute how likely the data is given that u (blue curve) and how likely that u is in the population (orange curve). We multiply these and sum up the area under the product. In LMM, the product of two normals is still normal — closed form. In GLMM, p(y|u) is a logistic or Poisson, and the product has no simple form. We must approximate numerically.
5
How Software Solves It

Three approximation strategies

Because the GLMM integral has no closed form, software uses one of three numerical strategies. Here's what each does and when to use it:

GH
Gauss-Hermite Quadrature
Approximates the integral by evaluating the integrand at a fixed set of carefully chosen points (quadrature points) and summing up the weighted values. More quadrature points → more accurate, but slower. The gold standard for accuracy when feasible.
Limitation: Computational cost grows exponentially with the number of random effects. Fine with 1–2 random effects; impractical with 10+.
✓ Best accuracy (low-dimensional)
LA
Laplace Approximation
Instead of computing the integral exactly, Laplace's method approximates the integrand with a normal distribution centred at its mode (peak). The integral of a normal is easy to compute analytically.
Intuition: Near its peak, almost any smooth function looks like a bell curve if you zoom in. Laplace approximation capitalises on this. The approximation is exact when the integrand truly is normal — which it is for LMMs but only approximately for GLMMs.
✓ Fast, widely used (glmer default)
PQL
Penalised Quasi-Likelihood
PQL reformulates the GLMM as a weighted normal mixed model with a "working response" — a linearised approximation of the actual outcome. It then repeatedly refits this normal LMM (hence "doubly iterative") until convergence.
Limitation: PQL can produce biased estimates, especially for binary outcomes with low event rates or small clusters. Generally not recommended for modern analyses unless the dataset is very large.
⚠ Use with caution — bias risk
What this means in practice for your research

In R, glmer(remission ~ phq8 + (1|patient), family=binomial) uses Laplace approximation by default. For more accurate inference, use nAGQ=5 or higher (Gauss-Hermite with 5 quadrature points). If your model has many random effects or fails to converge, consider Bayesian alternatives (Stan/brms) which use MCMC to sample the exact posterior rather than approximating it.

Where GLMMs appear in the RADAR-CNS pipeline

In Phase 5 (ML classification), the outcome is binary: PHQ-8 ≥ 10 vs <10. If you wanted to do this statistically rather than with a machine learning classifier, a GLMM with logistic link would be the right tool — it would give you interpretable odds ratios for each acoustic feature, properly accounting for repeated measures. The ML classifier is more flexible but less interpretable. Both have their place.
